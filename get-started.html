开始之前

一些必须知道的概念:

MVC

wula的MVC不是传统意义上的模型(Model)-视图(View)-控制器(Controller),而是模块(Module)-视图(View)-控制器(Controller).

除了模块，wula还提供基于插件(Plugin)机制的扩展(Extension)功能。

对修改关闭，对扩展开放是wula遵守的第一原则.

至于数据库访问:

在wula中数据库的访问使用封装的Table,View,SimpleTable类进行， 或者直接通过App::db()或App::table()进行数据库操作，我们将在数据库文档中详细说明每一个数据库访问方式.

模块(Module)

在wula中将一些业务紧密相关，不适合再细分的功能组合成一个自包含的整体称之为模块，各模块间不应有业务逻辑上的耦合。 模块可以通过插件机制为其它模块或扩展来动态的增加自身功能。模块也可以对外提供接口供其它模块使用， 当一个模块使用了另一个名多个模块提供的接口，那么他们之前便产生了依赖。

在wula中一个模块就是wwwroot/modules路径下的一个目录，模块的文件组成如下(wula目录结构)：

wwwroot
|-- modules
    |-- module_dir
        |-- classes
        |-- controllers
        |-- views
        |-- bootstrap.php
  1. classes 模块类文件放在此目录,非必须。
  2. controllers 模块控制器放在此目录，非必须。
  3. views 视图(模板)文件放在此目录，非必须。
  4. bootstrap.php 模块引导文件，此文件必须.
